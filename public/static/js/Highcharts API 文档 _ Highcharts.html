Highcharts 基本组成

名词解释

  • lang：语言文字对象，所有Highcharts文字相关的设置，【查看教程】
  • chart：图表区、图形区和通用图表配置选项，【查看教程】
  • colors：图表数据列颜色配置，是一个颜色数组，【查看教程】
  • credits: 版权信息，Highcharts在图表的右下方放置的版权信息及链，【查看教程】
  • drilldown：钻取，向下钻取数据，深入到其中的具体数据，【查看教程】
  • exporting：导出模块，导出功能配置，导出即将图表下载为图片或打印图表
  • legend：图例，用不同形状、颜色、文字等 标示不同数据列，通过点击标示可以显示或隐藏该数据列，【查看教程】
  • loading：加载中，加载选项控制覆盖绘图区的加载屏的外观和文字
  • navigation：导航，导出模块按钮和菜单配置选项组
  • noData：没有数据，没有数据时显示的内容
  • pane：分块，针对仪表图和雷达图专用的配置，主要设置弧度及背景色
  • plotOptions：针对不同类型图表的配置
  • series：数据列，图表上一个或多个数据系列，比如图表中的一条曲线，一个柱形，【查看教程】
  • title：标题，包括即标题和副标题，其中副标题为非必须的，【查看教程】
  • tooltip：数据点提示框，当鼠标滑过某点时，以框的形式提示改点的数据，比如该点的值，数据单位等，【查看教程】
  • Axis：坐标轴，包括x轴和y轴。多个不同的数据列可共用同一个X轴或Y轴，当然，还可以有两个X轴或Y轴，分别显示在图表的上下或左右，【查看教程】

  • line：直线图
  • spline：曲线图
  • area：面积图
  • areaspline：曲线面积图
  • arearange：面积范围图
  • areasplinerange：曲线面积范围图
  • column：柱状图
  • columnrange：柱状范围图
  • bar：条形图
  • pie：饼图
  • scatter：散点图
  • boxplot：箱线图
  • bubble：气泡图
  • errorbar：误差线图
  • funnel：漏斗图
  • gauge：仪表图
  • waterfall：瀑布图
  • polar：雷达图
  • pyramid：金字塔

accessibility

无障碍功能相关的配置。

chart

关于图表区和图形区的参数及一般图表通用参数。

colors

图表序列（Series）的默认颜色数组。即图表的第 n 个序列的颜色是该数组的第 n 个值。当序列的数量超过颜色数组的长度，后续的序列将会重复调用该数组里的值。默认值是： 

colors: ['#7cb5ec', '#434348', '#90ed7d', '#f7a35c', '#8085e9', 
   '#f15c80', '#e4d354', '#2b908f', '#f45b5b', '#91e8e1']
默认颜色还可以设置在序列及某种类型图表序列里，详见column.colors, pie.colors.

在 样式模式中，colors选项不存在。 相反，颜色在CSS中定义，并通过数据列或点的类名称应用，或者通过 chart.colorCount 设置。

历史版本

在Highcharts 3.x 中，默认值是： 

colors: ['#2f7ed8', '#0d233a', '#8bbc21', '#910000', '#1aadce', 
   '#492970', '#f28f43', '#77a1e5', '#c42525', '#a6c96a']

在Highcharts 2.x 中，默认值是： 

colors: ['#4572A7', '#AA4643', '#89A54E', '#80699B', '#3D96AE', 
   '#DB843D', '#92A8CD', '#A47D7C', '#B5CA92']

对于导出功能模块的详细文档，请参考 文档。

global

全局配置参数是针对所有 Highcharts 图表生效的配置，所以只能通过 Highcharts.setOption 函数来配置，例如： 
Highcharts.setOptions({
	global: {
		useUTC: false
	}
});

labels

设定可以放置在图表中任意位置的 HTML 标签。

lang

语言文字对象，Highcharts 图表中所有相关的文字都可以在这里设置，另外该配置是全局配置，针对所有图表有效，所有不能单独设置在某个图表中在，只能在图表初始化之前通过 Highcharts.setOptions 来设置生效。下面是一个实例： 
Highcharts.setOptions({
	lang: {
		months: ['一月', '二月', '三月', '四月', '五月', '六月',  '七月', '八月', '九月', '十月', '十一月', '十二月'],
		weekdays: ['星期日', '星期一', '星期二', '星期三', '星期四', '星期五', '星期六']
	}
});

legend

图例说明是包含图表中数列标志和名称的容器。各数列（若饼图则为各点）由图例中的标志和名称表示。

支持创建 自定义图例标志。

loading

加载选项控制着图表覆盖区域载入画面的显示效果。只有当chart.showLoading()被调用之后，该画面才会出现。 当需要展示和后台通信时，这个功能就变得很实用了。比如当通过XHR连接接收到新数据时。"loading"文本本身并不被包含在这个配置对象中，但是却被包含在lang对象中。

navigation

显示在导出模块中的按钮和菜单的选项集合。

noData

当图表没有数据时，图表中会显示相应的文字说明。该功能需要额外的引入 http://cdn.hcharts.cn/highcharts/modules/no-data-to-display.js 功能模块，其中显示的文字是在 lang.noData 中设置的。

没有数据的饼图，没有数据的直线图

pane

Applies only to polar charts and angular gauges. This configuration object holds general options for the combined X and Y axes set. Each xAxis or yAxis can reference the pane by index.

plotOptions

数据列配置是针对所有数据列及某种数据列有效的通用配置。

数据列的配置有三个级别：

  • 配置在 plotOptions.series，针对所有图表类型有效
  • 配置在 plotOptions.<数据列类型>，针对某种数据列有效
  • 配置在 series，针对某个数据列有效

上述三个级别的配置精准度越来越高，也就是越精准的配置会覆盖前面的配置

下面是实例代码及说明：

plotOptions: {
  spline: {           // 针对 spline 有效的配置
    lineWidth: 1 
  },
  series: {           // 针对所有数据列有效的配置
    lineWidth: 2
  }
}

series: [{
  id: 'series 1',
  type: 'spline',       // type 默认值是 'line'
  data: [1, 4, 56, 69],
  lineWidth: 2          // 该参数会覆盖 plotOptions.spline.lineWidth 里的配置
}, {
  data: [4, 5, 6],
  lineWidth: 3          // 该参数会覆盖 plotOptions.series.lineWidth 里的配置
}, {
  data: [10, 500, 199]  // 该数据列会继承 plotOptions.series.lineWidth 里的配置
}]

responsive

通过设定不同的响应规则来实现对图表在不同尺寸下的响应（图表中的每个组件都支持响应）。

series

图表的数据列，可以在每个数据列中指定其参数，另外也可以通过数据列配置（ plotOptions） 来指定对某一种类型的数据列有效的配置。

数据列和数据列配置（ plotOptions）选项不同之处在于数据列里的对象是图表中具体的某一个数据列，包括数据、类型、ID、下标等具体的值，而数据列配置只是针对某一种数据列有效的通用参数，下面是代码示例说明：

plotOptions: {
  spline: {           // 针对 spline 有效的配置
    lineWidth: 1 
  },
  series: {           // 针对所有数据列有效的配置
    lineWidth: 2
  }
}

series: [{
  id: 'series 1',
  type: 'spline',       // type 默认值是 'line'
  data: [1, 4, 56, 69],
  lineWidth: 2          // 该参数会覆盖 plotOptions.spline.lineWidth 里的配置
}, {
  data: [4, 5, 6],
  lineWidth: 3          // 该参数会覆盖 plotOptions.series.lineWidth 里的配置
}, {
  data: [10, 500, 199]  // 该数据列会继承 plotOptions.series.lineWidth 里的配置
}]

subtitle

图表副标题

title

图表标题。

tooltip

数据提示框指的当鼠标悬停在某点上时，以框的形式提示该点的数据，比如该点的值，数据单位等。数据提示框内提示的信息完全可以通过格式化函数动态指定；通过设置 tooltip.enabled = false 即可不启用提示框。

教程地址：https://www.hcharts.cn/docs/basic-tooltip

xAxis

X 轴（或分类轴）。默认情况下，x轴显示在图表的底部，y轴显示在左侧（多个y轴时可以是显示在左右两侧），通过设置chart.inverted = true 可以让x，y轴显示位置对调。

在多轴中，xAxis 可以配置成数组的形式。

X 轴可以通过 坐标轴对象 来进行动态更新或进行其他操作。

教程地址：https://www.hcharts.cn/docs/basic-axis

yAxis

Y 轴（或值轴）。默认情况下，x 轴显示在图表的底部，y轴显示在左侧（多个y轴时可以是显示在左右两侧），通过设置chart.inverted = true 可以让x，y轴显示位置对调。

在多轴中，yAxis 可以配置成数组的形式。

Y 轴可以通过 坐标轴对象 来进行动态更新或进行其他操作。

教程地址：https://www.hcharts.cn/docs/basic-axis

zAxis

Z 轴（或景深轴）

Z 轴可以通过 坐标轴对象 来进行动态更新或进行其他操作。

在线实例：

3D 散点图，3D 散点图

Axis

每个图表可以包含 0 个（例如饼图）到多个坐标轴。在常见的单序列笛卡尔图表中包含一个 X 轴和一个 Y 轴，在 3D 图表中还存在 Z 轴。

X 轴对象可以通过 chart.xAxis 获取，它是一个坐标轴对象数组，如果图表中只有一个数组，可以通过 chart.xAxis[0] 来获取，对应的多个 X 轴时是通过下来表获取对应坐标轴对象。同样的 Y 轴也是这样的。

如果需要通过数据列对象获取坐标轴对象，可以通过 series.xAxis 或 series.yAxis 来获取，注意这个对象并不是一个数组，而是一个坐标轴对象，因为一个数据列只能对应一个 X 轴和 Y 轴。

另外一种获取坐标轴对象的方法是通过配置坐标轴的 id，然后通过 chart.get(id) 来获取。

坐标轴对象的所有配置详见 xAxis 、yAxis 及 zAxis

Chart

图表对象是页面中代表单个图表的 JavaScript 对象。当通过图表构造函数 Highcharts.Chart() 来创建图表时，该函数返回的对象就是图表对象。

var chart1 = new Highcharts.Chart(options);

Element

SVG 元素是 JavaScript 封装的 SVG 对象，也就是对应的 Highcharts 中的各个元素。通过和 Renderer 对象一起使用，可以自由的在图表中甚至是不需要创建图表的网页中创建 SVG 元素。

Highcharts

Highcharts 的命名空间，所有的相关变量或方法都可以通过 Highcharts 来获取。 

var chart1 = new Highcharts.Chart(options);

Point

数据点对象是代表图表中的每个数据点的 JavaScript 对象。

数据点对象可以通过多种方式获取，包括在数据点事件回调函数里的 this；在数据列里所有的数据点可以通过 series.data 数据获得；

另外一种方式就是通过配置在数据点的 id，并通过 chart.get(id) 方法来或者这个数据点对象。

Renderer

绘图工具可以直接操作 Highcharts 的渲染层并绘制基础的图形，例如直接在图表中添加 圆形、矩形、路径或文字，甚至是独立于图表的图形。

在主流的浏览器中绘图工具对象是 SVG 的封装，对应的在低版本 IE （IE8 以下）中是 VML 封装

已经实例化的图表对象可以通过 chart.renderer 来获取绘图工具对象，如果需要独立图表的绘图工具对象，请通过 var renderer = new Highcharts.Renderer(parentNode, width, height); 来创建，其中 parentNode 是 HTML 元素（DOM）

绘图工具的方式支持链式调用，即你可以在创建绘图工具对象后在一个语句中依次调用 attr 、 css 、 add 等。

在线例子：给图表添加标注、基础绘图

Series

数据列对象是 JavaScript 中代表每条线、每个面积数据列、每个饼图等的对象

数据列对象可以通过多种方式获取。所有的数据列及数据点的事件回调函数里都有相对应的数据列关联对象； chart 对象里的 series 属性包含图表中所有数据列数组，数据点 point 对象也有同样的属性关联着数据列。

另外一种获取数据列的方式是通过它的 id 来获取，在配置时给定数据列的 id 并通过 chart.get(id) 方法来获取数据列对象。

数据列可以通过三个级别的配置来指定：1、针对所有数据列有效的配置是在 plotOptions.series 里； 2、针对某一种数据列类型有效的配置是配置在 plotOptions 下对应是数据列类型里，例如 plotOptions.line 里的配置只针对直线型有效； 3、针对某一个数据列有效的配置是在 series 里配置，也就是说 plotOptions 里的属性完全可以配置在 series

上述三个级别的配置的权重依次递增，也就是后面的（或者说更具体精准的）配置会覆盖前面的配置。

Legend

图例对象，可以通过 chart.legend 来获取，另外每个图表只有一个图例。
